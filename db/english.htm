A plain language description of the Welcome to OntoRave model

Section

Ο 1. Each Section may have a Name, which carries a String value.
Ο 2. Each Section may have a Created At, which carries a DateTime value.
Ο 3. Each Section may have a Updated At, which carries a DateTime value.
Ο 4. Each Section may have a Sort Code, which carries a int value.
Ο 5. Each Section may have a Body, which carries a String value.

Attribute

Ο 6. Each Attribute may have a Name, which carries a String value.
Ο 7. Each Attribute may have a Created At, which carries a DateTime value.
Ο 8. Each Attribute may have a Updated At, which carries a DateTime value.
Ο 9. Each Attribute may have a Sort Code, which carries a int value.
Ο 10. Each Attribute may have a Value, which carries a String value.

Value

Ο 11. Each Value may have a Name, which carries a String value.
Ο 12. Each Value may have a Created At, which carries a DateTime value.
Ο 13. Each Value may have a Updated At, which carries a DateTime value.
Ο 14. Each Value may have a Sort Code, which carries a int value.

Item

Ο 15. Item refers collectively to Sections, Attributes, Values, InfoNodes. All things true of Item are also true of these.

Ο 16. Each Item may have a Name, which carries a String value.
Ο 17. Each Item may have a Created At, which carries a DateTime value.
Ο 18. Each Item may have a Updated At, which carries a DateTime value.
Ο 19. Each Item may have a Sort Code, which carries a int value.

Ο 20. Each Item can have zero or more property_attrs (Property).

Info Node

Ο 21. Each Info Node may have a Name, which carries a String value.
Ο 22. Each Info Node may have a Created At, which carries a DateTime value.
Ο 23. Each Info Node may have a Updated At, which carries a DateTime value.
Ο 24. Each Info Node may have a Sort Code, which carries a int value.
Ο 25. Each Info Node may have a View, which carries a String value.

Ο 26. Each Info Node can have zero or more items (Item, a Section or Attribute or Value or InfoNode).

Ο 27. Info Nodes may have zero or more ref_targets (Item, a Section or Attribute or Value or InfoNode). Also, each Item can have zero or more ref_sources (InfoNodes).

Tag

Ο 28. Each Tag may have a Name, which carries a String value.

Ο 29. Tags may have zero or more items (Item, a Section or Attribute or Value or InfoNode). Also, each Item can have zero or more tags (Tags).

Property

Ο 30. Each Property may have a Name, which carries a String value.
Ο 31. Each Property may have a Value, which carries a String value.
